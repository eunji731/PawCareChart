# 🐾 멍케어차트(PawCare Chart) 프로젝트 통합 기획서 V2

## 1. 개요 및 비전
- **서비스명**: 멍케어차트 (PawCare Chart) - 반려견 통합 케어 대시보드
- **한 줄 정의**: 반려견의 병원 방문 기록, 진료비, 투약 정보 및 증상 통계를 한 곳에서 관리하는 **데이터 중심 케어 다이어리**
- **기획 모토**: "번거로운 기록은 짧게, 우리 아이 건강 관리는 깊게"

## 2. 프로젝트 배경 및 목적 (Single Source of Truth)
### 2.1. 만들게 된 계기
- 반려견을 키우다 보면 병원 기록, 진료비 영수증, 약 봉투 등이 여러 곳에 흩어져 관리가 어렵다.
- 과거에 어떤 증상으로 내원했었는지, 주기적인 구충은 언제 했는지 기억에 의존하는 한계를 해결하고자 한다.
- 복잡한 커뮤니티 기능 대신, **내 아이의 건강 데이터 관리**라는 본질에만 집중한 가벼운 웹앱이 필요했다.

### 2.2. 주요 목적
- 반려견의 진료/기록을 일자별, 목적별로 체계적으로 아카이빙
- 진료비 지출 내역을 데이터화하여 가계 경제 흐름 파악
- 영수증, 처방전, 증상 사진 등 시각적 보조 자료 보관
- **[Upgrade]** 수집된 데이터를 분석하여 예방케어 시점 및 건강 상태 예측

## 3. 핵심 기능 (Features)

### 3.1. 기본 관리 기능 (CRUD Core)
- **회원가입/로그인**: 사용자별 데이터 격리 및 보안 (JWT 기반)
- **반려견 등록**: 사진, 나이, 체중 등 기본 프로필 관리
- **병원/비용 기록**: 진료 내용과 결제 금액을 등록/수정/삭제
- **멀티미디어 보관**: 영수증 이미지 및 이상 징후 사진 업로드

### 3.2. [V2] 7대 고도화 분석 기능 (Enhanced Dashboard)
1. **🚨 예방케어 신호등**: 심장사상충, 정기 구충 등의 주기를 분석해 대시보드에 신호등(녹색/노란색/빨간색)으로 표시
2. **💊 처방약 복약 트래커**: 남은 약 복용 일수와 진행률을 바(Bar) 형태로 시각화하여 빼먹지 않게 지원
3. **📊 지출 예측 탭**: 최근 6개월 지출 추이를 차트로 분석하고, 다음 달 예상 고정 지출액 산출
4. **🏆 증상 워스트 Top 3**: 기록된 진료 데이터에서 가장 빈번한 증상 키워드를 추출하여 랭킹 제공
5. **🔁 반복 기록 원클릭 추가**: 똑같은 진료나 정기 결제 내역을 클릭 한 번으로 새 기록에 복사
6. **📝 빠른 관찰 메모 (Health Logs)**: 병원 가기 전 발견한 이상 징후(예: 구토, 무른 변)를 스냅샷 형태로 즉각 메모
7. **📢 월간 케어 브리핑**: "이번 달은 내원 2회, 전월 대비 지출 15% 감소" 등 한 달 요약 리포트 제공

## 4. 메뉴 구성 (Site Map)
- **홈 (Home)**: 신호등 대시보드, 이번 달 요약, 지출 차트, 최근 기록 통합 뷰
- **반려견 정보**: 상세 프로필 및 건강 히스토리 요약
- **케어 통합 기록**: 통합 리스트(병원+비용), 상세 보기, 등록/수정 모달
- **통계/분석**: 증상 워스트 랭킹, 지출 추이 상세 분석
- **마이페이지**: 내 정보 관리 및 데이터 백업 설정

## 5. 확정 기술 스택 (Technical Stack)

### Backend
- **Core**: **Spring Boot 3.x**, Java 17
- **Database**: **Supabase (PostgreSQL)**
- **ORM**: **Spring Data JPA** (Entity 기반 연관관계 매핑)
- **Auth**: Spring Security + JWT (Stateless)
- **Build**: Gradle

### Frontend
- **Core**: React 18 (TypeScript), Vite
- **Styling**: **Tailwind CSS** (다꾸 감성의 따뜻한 UI 라이브러리)
- **HTTP**: Axios
- **Deploy**: Vercel/Railway + Docker

## 6. DB 테이블 매핑 전략 (Data Domain)
- `users` / `dogs`: 회원 및 반려동물 기본 정보
- `vet_records`: 진료 일시, 병원명, **복약 기간/시작일** (트래커용) 
- `expense_records`: 결제 금액, 결제 수단, 영수증 URL
- `schedules`: **[NEW]** 다음 사상충약/접종 예정일 (신호등 대시보드 소스)
- `health_logs`: **[NEW]** 관찰용 빠른 메모 (이상 징후 사전 로그)
- `symptom_tags`: **[NEW]** 증상 정규화 키워드 (통계/랭킹 집계용)

## 7. 기대효과
- **보호자 측면**: 의료 데이터의 객관적 기록을 통해 과잉 진료 방지 및 체계적 예방 케어 가능
- **개발자 측면**: 실무와 유사한 REST API 설계, JPA 연관관계 매핑, JWT 보안, 외부 스토리지(Supabase) 연동 등 풀스택 역량 강화
