# Dom
- DOM은 브라우저가 HTML을 읽고 만들어 놓은 “화면용 객체 트리”
- 태그들의 부모-자식 구조로 정리된 것
- 자바스크립트가 실제 화면 요소를 조작할 수 있게 해주는 브라우저 내부 구조

## DOM 필요성
- 브라우저는 HTML을 읽은 뒤에 각 태그를 객체처럼 다룰 수 있는 구조로 바꿈
- 이 구조를 DOM이라고 함
- 자바스크립트가 이 DOM을 조작해서 화면을 바꿈
- 예시) 특정 태그 찾기, 글자 바꾸기, 버튼 숨기기, 클래스 추가하기, 새로운 요소 넣기, 클릭 이벤트 연결하기

### `노드(node)`
- DOM을 구성하는 각각의 조각
- HTML 태그 하나하나가 하나의 노드
- 텍스트, 속성, 주석 등도 모두 노드
- 노드들이 모여서 트리 구조를 이룸

### `노드 트리`
- DOM은 트리 구조로 되어 있음
- 부모 노드, 자식 노드, 형제 노드 등으로 구성됨
- 트리 구조이기 때문에 부모-자식 관계를 통해 요소를 찾거나 조작할 수 있음

### HTML vs DOM
- HTML : 문서에 적힌 원본 마크업
- DOM : 브라우저가 그 HTML을 해석해서 만든 메모리 속 구조

### 리액트 - DOM
- 리액트도 결국 최종적으로는 브라우저 DOM에 화면을 보여줘야함
- 전통 JS : HTML -> 브라우저 DOM 직접 조작(개발자가 직접 DOM을 많이 조작)
- 리액트 : HTML -> 리액트 가상 DOM -> 실제 브라우저 DOM(`상태(state)`가 바뀌면 리액트가 알아서 화면을 다시 계산해서 DOM에 반영하는 방식)
