# 🐾 멍케어차트(PawCare Chart) 통합 원페이저 v4

## 1. 프로젝트 개요
- **서비스명**: 멍케어차트 (PawCare Chart)
- **한 줄 소개**: 반려견의 병원 방문, 지출, 복약, 일정, 관찰 메모를 한 곳에서 기록하고 관리하는 개인용 케어 기록 웹앱
- **서비스 성격**: 커뮤니티형 서비스가 아니라, **내 반려견의 건강 데이터 관리**에 집중한 가벼운 기록형 서비스

## 2. 만들게 된 계기
반려견을 키우다 보면 병원 방문 기록, 진료 내용, 지출 내역, 영수증, 약 정보가 여러 곳에 흩어져 남는다.
시간이 지나면 “언제 병원에 갔는지”, “무슨 증상으로 진료를 받았는지”, “얼마를 썼는지”, “약은 며칠치였는지”를 다시 찾기 어렵다.

멍케어차트는 이런 정보를 한 곳에 모아 **쉽게 기록하고 다시 찾을 수 있도록** 하기 위해 기획했다.

## 3. 핵심 목적
- 반려견의 케어 기록을 일자별로 체계적으로 관리
- 진료비, 약값, 검사비 등 지출 흐름을 정리
- 영수증, 처방전, 사진 같은 첨부자료를 함께 보관
- 복약 기간, 예방 케어 일정, 관찰 메모까지 함께 관리
- 누적 데이터를 바탕으로 월간 요약과 기본 분석 기능 제공
- 공통코드 기반으로 선택값을 표준화해 유지보수성을 높임

## 4. 핵심 개념
멍케어차트는 기존의 **병원 기록**과 **지출 기록**을 완전히 별도 메뉴로 나누는 대신,
사용자 경험상 하나의 흐름인 **케어기록**으로 통합해 관리한다.

다만 병원 기록과 지출 기록은 성격이 조금 다르므로,
등록/수정 화면에서 **기록 유형(type)** 을 먼저 선택하고 선택한 유형에 따라 폼을 다르게 보여준다.

### 기록 유형 예시
- **MEDICAL**: 병원 방문, 진단, 처치, 복약 정보 중심
- **EXPENSE**: 지출, 영수증, 비용 메모 중심

## 5. UI/UX 방향
### 5.1. 메뉴 구조
- **홈**: 이번 달 요약, 최근 기록, 예방케어 알림, 복약 진행 상태
- **반려견 정보**: 기본 프로필, 몸무게, 대표 사진, 건강 이력 요약
- **케어기록**: 병원/지출 통합 목록, 상세, 등록/수정/삭제
- **마이페이지**: 사용자 정보, 비밀번호 변경, 로그아웃

### 5.2. 케어기록 화면 구성
- 상단 필터: **전체 / 병원 / 지출**
- 등록 버튼: **기록 추가**
- 목록: 날짜순 통합 정렬
- 상세: 공통 정보 + 유형별 세부 정보

### 5.3. 케어기록 등록 방식
1. 반려견 선택
2. 기록 유형 선택
   - 병원 기록
   - 지출 기록
3. 유형별 폼 노출

### 5.4. 유형별 폼 예시
#### 병원 기록(MEDICAL)
- 방문일
- 병원명
- 증상
- 진단명
- 처치 내용
- 복약 시작일
- 복약 일수
- 첨부파일

#### 지출 기록(EXPENSE)
- 지출일
- 카테고리
- 금액
- 메모
- 첨부파일
- 관련 병원 기록 연결(선택)

## 7. 데이터 설계 방향
### 7.1. 핵심 원칙
- UI는 **케어기록 통합형**으로 설계
- 데이터는 공통 정보와 유형별 상세 정보를 분리해 관리
- 첨부파일은 공통 구조로 연결
- 선택형 값은 **공통코드(code)** 로 관리

### 7.2. 구조 요약
- `code_groups`, `codes`: 공통코드 관리
- `care_records`: 케어기록 공통 헤더
- `care_record_medical_details`: 병원 기록 전용 상세 정보
- `care_record_expense_details`: 지출 기록 전용 상세 정보
- `files`, `file_mappings`: 첨부파일 공통 관리
- `schedules`: 일정 관리
- `health_logs`: 관찰 메모
- `symptom_tags`: 증상 통계용 태그

### 7.3. 공통코드 적용 대상 예시
- 기록 유형: `MEDICAL`, `EXPENSE`
- 지출 카테고리: `CONSULTATION`, `MEDICINE`, `EXAM`, `VACCINE`, `GROOMING`, `FOOD`, `ETC`
- 일정 유형: `VACCINE`, `HEARTWORM`, `CHECKUP`, `GROOMING`, `MEDICATION`, `ETC`
- 증상 강도: `LOW`, `MEDIUM`, `HIGH`
- 사용자 권한: `ROLE_USER`, `ROLE_ADMIN`

## 8. 기대 효과
### 사용자 측면
- 병원 기록과 지출 기록을 따로 찾지 않아도 된다
- 시간순으로 통합 관리되어 사용성이 높다
- 영수증, 사진, 진료 내용이 함께 남아 재조회가 쉽다
- 복약/예방케어까지 이어지는 관리가 가능하다

### 개발 측면
- 인증, 파일 업로드, 통합 CRUD, 유형 분기 UI를 모두 경험할 수 있다
- 공통 엔티티 + 상세 엔티티 구조 설계를 실습할 수 있다
- 공통코드 기반 선택값 관리 구조까지 포트폴리오로 정리할 수 있다
- 실제 서비스형 프로젝트 구조를 체계적으로 문서화하기 좋다
