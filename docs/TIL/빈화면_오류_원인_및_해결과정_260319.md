# 빈화면(White Screen) 오류 원인 및 해결 과정

## 1) 개요
- 발생/해결 일자: 2026-03-19
- 관련 커밋: `953d106` (`front fix : 빈화면만 출력되는 오류 해결`)
- 증상: 페이지 접속 시 UI가 렌더링되지 않고 흰 화면만 표시됨

## 2) 원인 분석
아래 원인은 커밋 변경점과 TypeScript/Vite 설정(`verbatimModuleSyntax: true`)을 기준으로 재구성했다.

### 원인 A (핵심)
- 파일: `src/pages/Home/components/MainDataTable.tsx`
- 기존 코드:
  - `import { Table, Column } from '../../../components/common/Table';`
- 문제점:
  - `Column`은 `Table.tsx`에서 `interface`(타입)로 선언된 항목이다.
  - 타입 전용 심볼을 값 import로 가져오면 모듈 해석 시점에 에러가 발생할 수 있다.
  - 이 경우 해당 모듈 로딩이 중단되고, 결과적으로 React 앱 마운트가 깨져 흰 화면으로 보일 수 있다.

### 원인 B (보조)
- 파일: `src/components/common/Tabs.tsx`
- 기존 코드:
  - `import React from 'react';`
- 문제점:
  - 현재 설정(`jsx: react-jsx`)에서는 React 기본 import가 필수 아님.
  - React 19 + ESM 환경에서 불필요/호환성 이슈가 있는 import는 모듈 로딩 실패 가능성을 높인다.

## 3) 해결 작업
커밋 `953d106`에서 다음과 같이 수정했다.

1. `MainDataTable.tsx` import 구문 분리
   - 변경 전: `import { Table, Column } from ...`
   - 변경 후:
     - `import { Table } from ...`
     - `import type { Column } from ...`
   - 효과: 타입 전용 심볼을 명시적으로 분리해 런타임 모듈 해석 오류를 차단.

2. `Tabs.tsx`의 불필요한 React 기본 import 제거
   - 변경 전: `import React from 'react';`
   - 변경 후: 해당 라인 삭제
   - 효과: JSX 자동 런타임 설정과 충돌 가능성 제거.

3. 부가 정리
   - `useSignup.ts`에서 미사용 변수(`response`) 제거
   - `index.html` 포맷 정리

## 4) 결과
- 수정 이후 페이지가 정상 렌더링되어 흰 화면 현상 해소.
- 핵심은 "타입/값 import 구분"이며, 동일 증상 재발 시 가장 먼저 해당 패턴을 점검해야 한다.

## 5) 재발 방지 가이드
1. 타입은 항상 `import type`으로 분리 (`verbatimModuleSyntax: true` 환경 필수 수칙).
2. React 17+ JSX 자동 런타임에서는 불필요한 `import React from 'react'` 사용 금지.
3. PR/커밋 전 체크:
   - TypeScript 에러 확인
   - 브라우저 콘솔의 `does not provide an export named ...` 계열 오류 확인
