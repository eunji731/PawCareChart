1. 프로젝트 개요
서비스명: 멍케어차트 (PawCare Chart)

한 줄 소개: 반려견의 과거 케어 기록(병원/지출/일상)과 미래의 일정(예약/접종)을 유기적으로 관리하는 개인용 펫 헬스케어 웹앱

서비스 성격: 다수의 사용자가 증상 태그 풀을 공유하며 집단지성을 형성하되, 핵심은 내 반려견의 건강 데이터의 체계적인 관리에 집중한 유틸리티 서비스

2. 만들게 된 계기
반려견을 키우다 보면 과거의 기록(병원 방문, 지출, 관찰 메모)과 미래의 계획(미용 예약, 예방접종 예정일)이 여러 곳에 흩어져 관리가 어렵다.
단순한 가계부를 넘어, "어떤 증상 때문에 병원을 예약했는지", "그 진료 결과로 얼마를 썼는지" 등 시간의 흐름에 따른 반려견의 헬스케어 여정을 한 곳에서 직관적으로 기록하고 추적하기 위해 기획했다.

3. 핵심 목적
반려견의 케어 기록(병원, 지출, 일상 메모)을 타임라인 형태로 체계적으로 관리

다가오는 일정(미용, 진료, 접종)을 분리하여 직관적으로 확인

**[일정 완료 ➔ 케어기록]**으로 이어지는 자연스러운 데이터 자동 전환 경험 제공

해시태그 형태의 다대다(N:M) 하이브리드 증상 태그 시스템으로 정확한 통계 및 쉬운 입력 지원

4. 핵심 개념
멍케어차트는 사용자의 목적에 따라 크게 **과거(케어기록)**와 미래(일정/예약) 두 개의 큰 축으로 나뉜다.

일정/예약 (Future): 다가올 이벤트와 그 사유(증상, 요청사항)를 미리 기록.

케어기록 (Past): 완료된 일정이나 일상적인 지출/메모를 통합 관리. 등록 시 **기록 유형(MEDICAL, EXPENSE, MEMO)**을 선택하여 하나의 뷰에서 필터링 조회.

5. UI/UX 방향
5.1. 메뉴 구조
홈: 오늘 다가온 일정, 이번 달 요약, 최근 증상 랭킹 브리핑

반려견 정보: 기본 프로필, 몸무게 추이, 건강 이력 요약

케어기록 (타임라인): 병원/지출/일상 통합 목록, 상세 내역, 기록 추가

일정/예약 (캘린더/D-Day): 캘린더 기반 일정 목록, 다가오는 예약 관리

마이페이지: 사용자 정보, 설정, 로그아웃

5.2. 일정/예약 등록 폼
미래의 방문 목적과 증상을 미리 기록하는 영역

날짜 및 시간

일정 유형 (진료, 미용, 예방접종 등)

제목 (예: 튼튼동물병원 방문)

상세 메모: 진료 예약 시 이상 증상 기입, 미용 예약 시 스타일 요청사항 기입

알림 설정

5.3. 케어기록 등록 방식 (하이브리드 태그 적용)
기록 유형 선택 (병원 / 지출 / 일상 메모)

공통 입력: 날짜, 제목, 노트, 사진 첨부

증상 태그 선택: 기존 유저들이 생성한 태그 풀에서 클릭하여 선택하거나, 새로운 태그 직접 입력

유형별 상세 폼 입력 (병원 상세 or 지출 카테고리/금액)

6. 주요 기능
6.1. 기본 (MVP) 기능
회원가입 / 로그인 / 로그아웃

반려견 등록 / 조회 / 수정

케어기록 타임라인 통합 조회 및 유형별 분기 등록

일정/예약 D-Day 관리 및 메모 기능

하이브리드 증상 태그 선택 및 커스텀 추가

첨부파일(사진/영수증) 업로드

6.2. 고도화 기능 (차별화 포인트)
일정 ➔ 기록 자동 전환: 지난 일정을 띄워주고 클릭 시 '케어기록' 폼으로 내용(증상 메모 등) 자동 이관

증상 랭킹(Symptom Rank): 누적된 다대다 증상 태그 기반 빈도수 통계

복약 트래커 및 예방 케어 자동 알림

월간 케어 브리핑 (지출 및 건강 요약)

7. 데이터 설계 방향
7.1. 핵심 원칙
UI는 타임라인으로 통합하되, DB는 부모(Record)와 자식(Details)으로 정규화하여 저장

증상 태그는 마스터 테이블과 N:M 매핑 테이블을 두어 데이터 파편화(오타 등) 방지

선택형 값은 **공통코드(code)**로 관리하여 프론트엔드 하드코딩 최소화

7.2. 구조 요약 (Schema v5 기준)
care_records: 케어기록 공통 헤더 (RECORD_TYPE: MEDICAL, EXPENSE, MEMO)

care_record_medical_details / care_record_expense_details: 유형별 1:1 상세

schedules: 일정/예약 관리 (증상 사전 메모 포함)

symptoms_master & care_record_symptoms: 글로벌 증상 사전 및 다대다 연결 태그

files, file_mappings: 첨부파일 공통 관리

codes, code_groups: 공통코드

8. 기대 효과
사용자 측면
"어떤 증상으로 예약했지?"를 달력에서 바로 확인하고, 다녀온 뒤 지출 내역으로 매끄럽게 연결된다.

매번 증상을 타자로 칠 필요 없이 칩(Chip) 클릭만으로 입력이 끝나고 정확한 랭킹 통계를 볼 수 있다.

일상 메모부터 무거운 병원비까지 시간순으로 한눈에 파악할 수 있다.

개발 측면
다대다(N:M) 관계 매핑과 JPA 영속성 전이(Cascade)를 깊이 있게 다뤄볼 수 있다.

슈퍼타입-서브타입 테이블 구조를 활용한 통합 목록 조회(View)를 실습할 수 있다.

다른 유저의 태그 데이터를 공유하는 하이브리드 검색/생성 로직을 구현해 볼 수 있다.

9. 기술 스택
Frontend: Next.js (React), TypeScript, Tailwind CSS

Backend: Spring Boot, Spring Security, JPA/Hibernate

DB / Infra: PostgreSQL (Supabase), Vercel(Front), 연동 스토리지

10. 한 줄 결론
멍케어차트는 미래의 일정(예약/증상)과 과거의 케어기록(진료/지출)을 유기적으로 연결하고, 하이브리드 태그 시스템을 통해 입력 편의성과 데이터 정합성을 동시에 잡은 완성형 펫 헬스케어 앱이다.
