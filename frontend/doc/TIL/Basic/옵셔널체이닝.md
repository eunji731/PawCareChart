# 옵셔널 체이닝

## 옵셔널 체이닝?
- 옵셔널 체이닝은 자바스크립트 문법 중 하나로, 중간 값이 null 또는 undefined여도 에러 없이 안전하게 접근하게 해주는 문법
```typescript
// 문법
?.

// 예제
obj?.prop
obj?.method()
arr?.[0]
```

## 사용이유
- 자바스크립트에서는 없는 값에 접근하면 에러 발생
- if 문으로 `null/undefined` 체크를 매번 해야 하는 번거로움
- 옵셔널 체이닝을 사용하면 에러대신 `undefined` 반환

```typescript
// 예제
if (error.response?.status === 401) {
}

// 동일의미
if (error.response && error.response.status === 401) {
}
// error.response가 있으면 .status를 본다
// error.response가 없으면 그냥 undefined로 처리
```

## 동작 원리
```javascript
const a = {
  b: {
    c: 10
  }
};
console.log(a?.b?.c); // 10

const a = null;
console.log(a?.b?.c); // undefined
```

## 안전범위
- 옵셔널 체이닝은 붙인 지점까지만 안전
```javascript
obj?.a.b
// 완전한 안전 X
// obj는 없을 수 있으니 안전하게 넘기지만, obj.a는 있다고 가정하고 .b를 보기 때문
// .b에 접근하려다 에러남

// 안전 ver
obj?.a?.b
```

## 메서드 호출 시 사용가능
```javascript
user.onClick?.()
// onClick이 있으면 호출, 없으면 undefined 반환(넘어감)

// 이전 방식
if (user.onClick) {
  user.onClick();
}
```

## 배열 접근 시 사용가능
```javascript
const arr = null;
console.log(arr?.[0]); // undefined

const arr = [10, 20, 30];
console.log(arr?.[0]); // 10
```

## &&와 차이
```javascript
// &&
const status = error && error.response && error.response.status;

// 옵셔널 체이닝
const status = error?.response?.status;

error.response?.status
// 에러 객체 안에 response가 있으면 status를 보고, 없으면 그냥 undefined로 처리해라
```

## 주의
- 옵셔널 체이닝은 에러를 감추는 문법이 아니라 없는 값에 안전하게 접근하는 문법
- 이 값이 없어도 정상인 상황”에서 써야 함
- 실제로는 꼭 있어야 하는 값인데 계속 ?.를 붙이면 문제가 생겨도 조용히 undefined가 되어버려서 원인 찾기 어려움

### 사용시 좋은 경우
- 네트워크 응답처럼 있을 수도 없을 수도 있는 값
- 선택적 데이터
- 렌더링 전 아직 로딩 안 된 데이터  

### 사용시 나쁜 경우
- 반드시 존재해야 하는 핵심 값
- 없으면 바로 오류를 잡아야 하는 값
