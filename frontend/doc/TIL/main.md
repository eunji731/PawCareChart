# main.tsx 분석

### main.tsx
- HTML과 React 앱을 연결하는 다리
`index.html`
- `<div id="root"></div>` 준비
- `main.tsx`가 root를 찾음
- `createRoot`로 리액트 앱 시작 준비
- `render(<App />)`
- `App.tsx`가 브라우저에 표시됨

- `App.tsx`가 최상위 앱 본체이고, `main.tsx`가 그걸 브라우저에 연결한다

### 전체 코드 흐름
1. 리액트 관련 기능 가져옴
2. 전역 CSS 불러옴
3. App 컴포넌트 불러옴
4. HTML 안의 root라는 영역을 찾음
5. 그 안에 <App />을 렌더링함
6. StrictMode로 감싸서 개발 중 점검을 더 엄격하게 함

### StrictMode
- 리액트에서 제공하는 기능(개발 검사 용도)
- 앱의 잠재적인 문제를 찾기 위해 렌더링 시 추가 검사를 하는 모드
- 개발 모드에서만 동작
- 프로덕션 빌드에는 포함되지 않음
- StrictMode로 감싸면 다음과 같은 추가 검사가 동작함
    - 컴포넌트의 렌더링이 의도치 않게 부작용을 일으키는지 확인
    - 레거시 API 사용 여부 확인
    - 예상치 못한 사이드 이펙트 확인
    - 오래된 생명주기 메서드 사용 여부 확인

### createRoot
- 리액트 앱을 브라우저 DOM에 붙이는 시작 함수
- 리액트 앱을 꽂아 넣을 자리와 엔진을 준비하는 함수
- 리액트 18부터 도입된 기능
- 기존의 ReactDOM.render()를 대체하는 기능
- createRoot()를 사용하면 다음과 같은 장점이 있음
    - 리액트 18의 새로운 기능(Concurrent Features)을 사용할 수 있음
    - 렌더링 성능이 향상됨
    - 메모리 사용량이 줄어듦
    - 개발 모드에서 더 엄격한 검사가 가능함

### 전역 CSS
- 앱 전체에 적용될 기본 스타일 파일을 불러오는 것
- 앱 전체 공통 스타일을 시작할 때 미리 로드

### App
- App.tsx를 가져오는 것
- main.tsx는 직접 여러 페이지를 구성하지 않지만 전체 앱 본체인 App을 불러와서 화면에 붙이는 역할

### `document.getElementById('root')`
- 브라우저의 실제 HTML 문서 안에서
id="root"인 요소를 찾는 것
- HTML에는 비어 있는 `<div id="root"></div>` 하나가 있고 main.tsx가 그걸 찾아서 거기에 `<App />`을 넣는 구조

### `!`
- TypeScript 문법
- 이 값은 null이 아닐 거라고 내가 확신한다

### `createRoot(...).render(...)`
- 브라우저의 root 영역에 App 컴포넌트를 그려라
- root DOM 요소를 찾고
- 그 DOM에 리액트 루트를 만들고
- 거기에 `<App />`을 렌더링한다

### render
- 리액트 컴포넌트를 실제 화면에 나타나게 하는 것
- `<App />`라는 리액트 컴포넌트 정의를 진짜 브라우저 화면에 보이게 만드는 단계
