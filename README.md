# PawCareChart (멍케어차트)

반려견의 **병원 기록, 지출 내역, 일상 메모, 일정**을 한 곳에서 관리하는 개인용 펫 케어 기록 웹앱입니다.

단순 메모나 가계부를 넘어서,  
**과거의 케어 기록**과 **앞으로의 일정/예약**을 연결해 반려견의 건강 흐름을 시간순으로 관리하는 것을 목표로 합니다.

---

## 1. 프로젝트 소개

반려견을 키우다 보면 병원 방문 내용, 진단 결과, 약 복용 정보, 영수증, 사진, 다음 예약 일정이 여러 곳에 흩어지기 쉽습니다.

PawCareChart는 이런 문제를 줄이기 위해 기획한 서비스로,

- 병원 기록
- 지출 기록
- 일상 메모
- 향후 일정
- 증상 태그
- 첨부파일

을 하나의 흐름 안에서 관리할 수 있도록 설계했습니다.

특히,

- **일정(Future)** 과 **케어기록(Past)** 을 분리하면서도
- 실제 사용 시에는 자연스럽게 이어지도록 구성하고,
- 반복 입력이 많은 증상 정보는 **태그 기반**으로 관리하여
- 입력 편의성과 데이터 정합성을 함께 잡는 방향으로 진행 중입니다.

---

## 2. 기획 배경

반려견 케어 기록은 생각보다 관리 포인트가 많습니다.

- 어떤 증상 때문에 병원에 갔는지
- 진단 결과가 무엇이었는지
- 약은 며칠치였는지
- 비용은 얼마나 들었는지
- 다음 예약은 언제인지
- 관련 사진이나 영수증은 어디에 저장했는지

기존에는 메모앱, 가계부, 사진첩, 캘린더 등에 흩어져 남기기 쉬워서 다시 찾기 번거롭습니다.

이 프로젝트는 이런 정보를 **한 화면 흐름 안에서 다시 찾기 쉽게 정리하는 개인용 서비스**를 만드는 것을 목표로 시작했습니다.

---

## 3. 핵심 목표

- 반려견의 케어 기록을 **타임라인 형태**로 관리
- 병원/지출/메모를 **통합된 기록 구조**로 조회
- 다가오는 일정과 예약을 **별도로 관리**
- 일정 완료 후 케어기록으로 자연스럽게 이어지는 구조 설계
- 증상 태그를 **공용 마스터 + N:M 연결 구조**로 관리
- 사진, 영수증, 처방전 등 파일 업로드 지원
- 월간 지출/건강 흐름을 한눈에 파악 가능하도록 확장

---

## 4. 주요 기능

### MVP 범위

- 회원가입 / 로그인 / 로그아웃
- 반려견 등록 / 조회 / 수정
- 케어기록 등록 / 목록 조회 / 상세 조회 / 수정 / 삭제
- 일정 등록 / 조회 / 관리
- 증상 태그 선택 및 신규 태그 추가
- 사진 / 영수증 / 처방전 업로드
- 월별 지출 조회

### 케어기록 구성

케어기록은 하나의 부모 테이블(`care_records`) 아래에서 유형별 상세 정보를 분리하는 구조입니다.

- `MEDICAL`: 병원 방문 기록
- `EXPENSE`: 지출 기록
- `MEMO`: 일상 관찰 기록

이 구조를 통해 UI에서는 통합 타임라인처럼 보이되, 저장은 정규화된 형태로 유지할 수 있도록 설계했습니다.

### 일정 관리

일정은 미래 시점의 이벤트를 기록합니다.

예시:

- 병원 예약
- 예방접종
- 재진 일정
- 미용 일정
- 복약 관련 일정

향후에는 일정에서 입력한 내용을 기반으로 케어기록 생성 시 일부 값을 자동 이관하는 흐름까지 고려하고 있습니다.

### 증상 태그

증상은 단순 문자열이 아니라,

- `symptoms_master`
- `care_record_symptoms`

구조로 설계해 다대다 관계로 관리합니다.

이 방식으로

- 오타로 인한 데이터 분산을 줄이고
- 반복 입력을 줄이며
- 증상 빈도 통계나 랭킹 기능으로 확장할 수 있도록 했습니다.

## 6. 데이터 설계 핵심

### 주요 테이블

- `users`: 사용자
- `refresh_tokens`: 인증 유지용 토큰 저장
- `dogs`: 반려견 정보
- `care_records`: 케어기록 공통 헤더
- `care_record_medical_details`: 병원 기록 상세
- `care_record_expense_details`: 지출 기록 상세
- `schedules`: 일정 관리
- `symptoms_master`: 증상 마스터
- `care_record_symptoms`: 케어기록-증상 연결
- `files`: 업로드 파일 정보
- `file_mappings`: 파일 대상 매핑
- `codes`, `code_groups`: 공통코드

### 설계 포인트

#### 1) 공통코드 기반 관리
선택형 값은 하드코딩 대신 공통코드로 관리합니다.

예:

- 사용자 권한
- 기록 유형
- 지출 카테고리
- 일정 유형
- 증상 심각도

#### 2) 통합 기록 + 상세 분리
조회는 통합적으로, 저장은 유형별로 분리하는 구조입니다.

- 공통 정보: 날짜, 제목, 노트
- 상세 정보: 병원명, 진단, 치료, 금액, 카테고리 등

#### 3) 파일 업로드 확장성
파일 자체 정보와 연결 대상을 분리하여,
향후 반려견 프로필 이미지, 케어기록 첨부, 기타 엔티티 연결까지 유연하게 확장 가능하도록 구성했습니다.

## 8. 디렉션 및 차별점

PawCareChart는 단순히 병원비를 기록하는 앱이 아니라,

- **왜 병원에 갔는지**
- **무슨 진단을 받았는지**
- **얼마가 들었는지**
- **다음 일정은 무엇인지**

를 하나의 흐름으로 이어서 관리하는 데 집중하고 있습니다.

차별화 포인트는 다음과 같습니다.

### 1) 과거 기록과 미래 일정의 연결
일정을 별도 관리하되, 완료 후 케어기록으로 연결되는 사용자 경험을 지향합니다.

### 2) 하이브리드 증상 태그 구조
단순 입력 필드가 아니라, 공용 증상 마스터를 활용해 정확한 집계와 빠른 입력이 가능하도록 설계했습니다.

### 3) 통합 타임라인 UX
병원/지출/메모를 각각 따로 보지 않고, 반려견의 건강 흐름을 한 곳에서 볼 수 있게 하는 것이 핵심입니다.
